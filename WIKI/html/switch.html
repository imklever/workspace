交换机

VLAN:

Trunk:

Trunk 模式下,允许多种Vlan tag的报文传送，trunk模式下具有native lan，默认为1.
当接收的报文没有vlan tag时，会自动加上 native vlan的tag，并在报文送出时，剥离native vlan tag
对于非native vlan 的其他vlan tag 值。不做vlan上的增加和剥离动作，仅根据规则，进行报文的接收或丢弃。

Access:

Access 模式下，接收或发送的报文必须都是无Vlan tag的报文。并在就收后，打上端口对应的PVID作为Vlan 值，以及在发送时剥离该Vlan值。
对于接收的含有Vlan信息的报文会直接进行丢弃（该动作为缺省动作，不确定是否有其他操作可选）。

试用情况:

Access 模式适用于交换机端口直连pc终端或服务器终端(该PC或服务器终端的IP地址配置不含Vlan tag)
Trunk 模式适用于交换机端口直连pc终端或服务器终端(该PC或服务器终端具有多个Vlan tag的IP 地址)
Trunk 模式适用于交换机和交换机之间的连接

命令，配置:

    enable

    configure terminal
    interface ip 1
        ip address 10.240.220.2 255.255.255.192
        vlan 533
        enable
        exit

    configure terminal
    interface port 2
        switchport mode trunk allow 1-123
        switchport mode access
        switchport access vlan 11

    show vlan
    vlan 11
    name c1
    enable
    member 1
    exit

    copy running-config startup-config
